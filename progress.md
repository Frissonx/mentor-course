День 7: reduce
- reduce(callback(acc, current), начальноеЗначение) — acc и current это параметры ОДНОГО коллбека, не разные аргументы reduce
- то, что вернул из коллбека → станет acc на следующем шаге (сердце reduce)
- начальное значение: 0 для суммы, 1 для произведения, '' для строк — рычаг под задачу
- ОСОЗНАННО опустить начальное → reduce стартует с первого элемента (нужно для max/min/склейки)
- День 7 ЗАКРЫТ: reduce полностью, включая reduce в объект
- reduce в число (сумма, произведение, max), в строку (склейка), в ОБЪЕКТ (счётчик/группировка)
- объектный reduce: acc[ключ] = (acc[ключ] || 0) + 1; return acc; начальное {}
- челлендж Дня 6 закрыт: filter→map→reduce = активные по городам

- День 10 (промисы):
- Promise = объект-"обещание" результата в будущем. 3 состояния: pending → fulfilled (resolve) ИЛИ rejected (reject)
- new Promise((resolve, reject) => {...}): resolve(значение)=успех, reject(ошибка)=провал
- .then(коллбек) — когда успех; .catch(коллбек) — когда ошибка (снова "передаю функцию, вызовут позже")
- .then цепляются плоско (вместо лесенки коллбеков = callback hell)

- День 10 (async/await) — ВЗЯТО:
- async/await = удобный синтаксис ПОВЕРХ промисов (не новый механизм)
- await = "подожди промис, дай результат в переменную"; работает ТОЛЬКО внутри async-функции
- await приостанавливает свою async-функцию, но не всю программу
